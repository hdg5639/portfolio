# 요구사항 반영표

## 기본 요구사항

| 요구사항 | 반영 내용 | 확인 위치 |
|---|---|---|
| 주제를 전달하는 텍스트와 이미지 | 자기소개, 취향, 프로젝트, TMI 콘텐츠와 로컬 SVG 이미지 사용 | 전체 페이지 |
| 버튼 클릭 리액션 | 인사, 좋아요, 밸런스 선택, 퀴즈 정답 처리 | `assets/js/app.js` |
| 팝업 창 | 인사 및 취향 상세 모달 | `index.html`, `lifestyle.html` |
| HTML + CSS + JS | 문서, 스타일, 동작을 각각 분리 | 전체 구조 |
| 바닐라 JS | 프레임워크 없이 DOM API와 Local Storage 사용 | `assets/js/app.js` |

## 심화 요구사항

| 요구사항 | 반영 내용 | 확인 위치 |
|---|---|---|
| 모바일 친화 기능 | 1050px, 800px, 560px 기준 반응형 레이아웃과 햄버거 메뉴 | `assets/css/style.css` |
| Git 코드 관리 | Git 저장소 초기화 및 단계별 3개 커밋, 사용 안내 추가 | `.git`, `GIT_GUIDE.md` |
| SPA를 MPA로 재구성 | 콘텐츠를 네 개의 HTML 문서로 분리 | 루트의 HTML 4개 |
| 동일한 네비게이션 | 네 페이지에 같은 헤더 마크업 사용 | 각 HTML 상단 |
| 로컬 링크 이동 테스트 | 모든 메뉴가 실제 `.html` 파일을 `href`로 연결 | 공통 네비게이션 |
| 게이미피케이션 | XP, 레벨, 진행률, 배지, 좋아요, 퀴즈 | 전체 페이지, 특히 `tmi.html` |
| 자유 기능 | 프로젝트 검색·필터, 밸런스 게임, 상태 저장, 접근성 대응 | 전체 프로젝트 |

## MPA 확인 방법

브라우저 주소 표시줄에서 메뉴를 이동할 때 파일명이 다음과 같이 바뀌면 MPA 이동이 정상입니다.

```text
index.html
lifestyle.html
projects.html
tmi.html
```
